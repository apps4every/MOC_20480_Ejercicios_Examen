Ejercicio No 62

You are developing an HTML5 web application for a surveyor company that displays topographic images.
The application must:

  • Display the topographic images at different zoom levels without loss of detail
  • Print the topographic images without loss of detail
  • Work from only one source file for each topographic image

You need to ensure that the topographic images display according to the requirements.
Which HTML5 element should you use?

  • SVG
  • CANVAS
  • SAMP
  • AREA

Correct Answer: A


Correct Answer: A
Section: Volume B
Explanation
Explanation/Reference:
Explanation:
SVG stands for Scalable Vector Graphics
SVG is used to define graphics for the Web
SVG is a W3C recommendation

The HTML svg element (introduced in HTML5) is a container for SVG graphics.

SVG has several methods for drawing paths, boxes, circles, text, and graphic images.
Reference:
https://www.w3schools.com/html/html5_svg.asp

Explicación

SVG: utiliza imágenes vectoriales, es el único formato que permite mantener la calidad sin importar la profundidad de zoom que se le aplique.

CANVAS: No sirve, aunque se utilice para crear gráficos, el resultado es un mapa de bits y este tipo de formatos pierde resolución al aplicarle zoom que es uno de los requisitos.

SAMP: No sirve ya que no se utiliza para mostrar gráficos sino texto que representan la salida de un ordenador, parecido a las etiquetas <code> o <var>

AREA: su objetivo es definir zonas seleccionables en una imagen. No cumple que mantenga la calidad independientemente del zoom que se le aplique.
